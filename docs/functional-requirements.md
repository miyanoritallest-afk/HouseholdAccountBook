# 機能要件・非機能要件・エラー仕様

[← 要件定義書に戻る](./requirements.md)

---

## 1. 機能要件

### 1-1. 認証機能

| # | 機能 | 説明 |
|---|------|------|
| A-1 | ユーザー登録 | メールアドレス・パスワードで新規登録 |
| A-2 | ログイン | メールアドレス・パスワードでログイン |
| A-3 | ログアウト | セッションを終了する |

### 1-2. 収支管理機能

| # | 機能 | 説明 |
|---|------|------|
| B-1 | 収支登録 | 種別（収入／支出）・金額・カテゴリ・日付・メモを登録 |
| B-2 | 収支編集 | 登録済みの収支情報を編集 |
| B-3 | 収支削除 | 登録済みの収支情報を削除 |
| B-4 | 収支一覧表示 | 月別に収支の一覧を表示 |
| B-5 | 月次集計表示 | 選択月の収入合計・支出合計・残高を表示 |

### 1-3. カテゴリ管理機能

| # | 機能 | 説明 |
|---|------|------|
| C-1 | カテゴリ一覧 | 収入／支出それぞれのカテゴリを一覧表示 |
| C-2 | カテゴリ登録 | 新しいカテゴリを追加 |
| C-3 | カテゴリ編集 | カテゴリ名を編集 |
| C-4 | カテゴリ削除 | カテゴリを削除 |

**デフォルトカテゴリ（支出）**: 食費 / 交通費 / 娯楽費 / 日用品 / 医療費 / 光熱費 / 通信費 / その他  
**デフォルトカテゴリ（収入）**: 給与 / 副業 / その他

### 1-4. グラフ・レポート機能

| # | 機能 | 説明 |
|---|------|------|
| D-1 | 支出カテゴリ別円グラフ | 月別の支出をカテゴリ別に円グラフで表示 |
| D-2 | 月別収支棒グラフ | 過去6ヶ月の収入・支出・残高を棒グラフで表示 |
| D-3 | レポート画面 | グラフと月次集計をまとめたレポートページ |

### 1-5. カレンダー機能

| # | 機能 | 説明 |
|---|------|------|
| E-1 | カレンダー表示 | ホーム画面に月カレンダーを表示し、収支データのある日付をハイライト |
| E-2 | 日付絞り込み | カレンダーの日付をクリックするとその日の収支のみ一覧に表示 |
| E-3 | 月ナビゲーション | 前月・翌月ボタンで表示月を切り替え |

### 1-6. 電卓 UI

| # | 機能 | 説明 |
|---|------|------|
| F-1 | インライン電卓 | 収支登録・編集画面の金額入力欄に電卓 UI をポップアップ表示 |
| F-2 | 四則演算 | 加算・減算・乗算・除算をサポート |
| F-3 | パーセント計算 | % キーで割合計算が可能 |
| F-4 | アクセシビリティ | フォーカストラップ・Escape で閉じる・スクリーンリーダー対応（aria-label）|

## 2. 非機能要件

### 2-1. セキュリティ

| # | 項目 | 要件 |
|---|------|------|
| S-1 | 認証 | ログイン済みユーザーのみ保護ページにアクセス可能 |
| S-2 | 認可 | 自分のデータのみ参照・操作可能（他ユーザーのデータへのアクセスは403エラー） |
| S-3 | パスワード保存 | パスワードはハッシュ化して保存（平文保存禁止） |
| S-4 | 通信 | 現在は HTTP 運用。独自ドメイン取得後に HTTPS 化予定（ACM + Route 53） |
| S-5 | CSRF対策 | CSRFトークンによる対策を実施 |
| S-6 | セッション管理 | セッション有効期限を設ける（24時間） |

### 2-2. パフォーマンス

| # | 項目 | 要件 |
|---|------|------|
| P-1 | 画面表示速度 | 収支一覧・グラフ表示は3秒以内 |
| P-2 | API レスポンス | 各 API は1秒以内にレスポンスを返す |

### 2-3. 信頼性・保守性

| # | 項目 | 要件 |
|---|------|------|
| R-1 | データ整合性 | 収支削除時にカテゴリとの整合性を保つ（外部キー制約） |
| R-2 | コード管理 | Git によるバージョン管理（GitHub） |
| R-3 | インフラ管理 | Terraform によるインフラのコード管理（IaC） |
| R-4 | デプロイ | `bash scripts/deploy.sh` による手動デプロイ（git pull → Docker ビルド → db:migrate） |

### 2-4. 対応環境

| # | 項目 | 要件 |
|---|------|------|
| E-1 | 対応ブラウザ | Google Chrome 最新版 |
| E-2 | 対応デバイス | PC のみ（レスポンシブ対応は対象外） |
| E-3 | 画面解像度 | 1280px 以上の横幅を推奨 |

## 3. エラー・例外仕様

### 3-1. 入力バリデーションエラー

| # | 対象 | 条件 | エラーメッセージ |
|---|------|------|----------------|
| V-1 | 金額 | 未入力 | 「金額を入力してください」 |
| V-2 | 金額 | 数値以外 | 「金額は数値で入力してください」 |
| V-3 | 金額 | 0以下 | 「金額は1円以上で入力してください」 |
| V-4 | 金額 | 10桁超（999,999,999円超） | 「金額は999,999,999円以下で入力してください」 |
| V-5 | 日付 | 未入力 | 「日付を選択してください」 |
| V-6 | カテゴリ | 未選択 | 「カテゴリを選択してください」 |
| V-7 | メモ | 200文字超 | 「メモは200文字以内で入力してください」 |
| V-8 | メールアドレス | 未入力 | 「メールアドレスを入力してください」 |
| V-9 | メールアドレス | 形式不正 | 「正しいメールアドレス形式で入力してください」 |
| V-10 | パスワード | 未入力 | 「パスワードを入力してください」 |
| V-11 | パスワード | 8文字未満 | 「パスワードは8文字以上で入力してください」 |
| V-12 | カテゴリ名 | 未入力 | 「カテゴリ名を入力してください」 |
| V-13 | カテゴリ名 | 20文字超 | 「カテゴリ名は20文字以内で入力してください」 |

### 3-2. 認証エラー

| # | 状況 | 表示内容・挙動 |
|---|------|---------------|
| E-1 | ログイン失敗（メール・パスワード不一致） | 「メールアドレスまたはパスワードが正しくありません」をフォーム上部に表示 |
| E-2 | 登録済みメールアドレスで新規登録 | 「このメールアドレスはすでに登録されています」を表示 |
| E-3 | 未ログイン状態で保護ページにアクセス | ログイン画面にリダイレクト |
| E-4 | セッション切れ | 「セッションの有効期限が切れました。再度ログインしてください」を表示しログイン画面へ |

### 3-3. データ操作エラー

| # | 状況 | 表示内容・挙動 |
|---|------|---------------|
| E-5 | 他ユーザーのデータへのアクセス | 403エラー「アクセス権限がありません」を表示 |
| E-6 | 存在しないリソースへのアクセス | 404エラー「データが見つかりませんでした」を表示 |
| E-7 | カテゴリ削除時に紐づく収支が存在する | 「このカテゴリには収支データが紐づいているため削除できません」を表示 |

### 3-4. システムエラー

| # | 状況 | 表示内容・挙動 |
|---|------|---------------|
| E-8 | サーバーエラー（500系） | 「システムエラーが発生しました。しばらく経ってから再度お試しください」を表示 |
| E-9 | ネットワークエラー | 「通信エラーが発生しました。ネットワーク接続を確認してください」を表示 |
