# 家計簿アプリ 要件定義書

| 項目 | 内容 |
|------|------|
| 作成日 | 2026-05-08 |
| バージョン | 1.1 |

---

## 1. プロジェクト概要

| 項目 | 内容 |
|------|------|
| アプリ名 | 家計簿アプリ（Zaim風） |
| 目的 | 個人の収入・支出を記録・管理し、家計の状況を可視化する |
| 参考アプリ | Zaim |

---

## 2. ターゲットユーザー

- 個人ユーザー（自分自身）
- 日々の収支を手軽に記録・管理したい人

---

## 3. 技術スタック

| レイヤー | 技術 |
|----------|------|
| フロントエンド | Next.js (React) |
| バックエンド | Ruby on Rails（API モード） |
| データベース | MySQL |
| 認証 | メール＆パスワード認証（devise 等） |
| グラフ | Recharts（予定） |

---

## 4. 技術選定理由

### フロントエンド: Next.js (React)

| 観点 | 理由 |
|------|------|
| 採用実績 | 国内外問わず広く採用されており、求人・学習リソースが豊富 |
| 開発効率 | コンポーネント設計により UI の再利用性が高い |
| ルーティング | ファイルベースルーティングで直感的にページ管理が可能 |
| 将来性 | Vercel によるホスティング・デプロイが容易 |

### バックエンド: Ruby on Rails（API モード）

| 観点 | 理由 |
|------|------|
| 開発速度 | 規約ベースの設計により CRUD 実装を高速に行える |
| エコシステム | devise（認証）・active_model_serializers（JSON整形）等の gem が充実 |
| 可読性 | Ruby の記法が読みやすく保守性が高い |

### データベース: MySQL

| 観点 | 理由 |
|------|------|
| 実績 | 国内 Web サービスでの採用実績が多く、知見が豊富 |
| Rails との親和性 | Active Record との組み合わせで安定した動作が期待できる |
| 集計クエリ | 月次集計・カテゴリ別集計に必要な GROUP BY / SUM 等が扱いやすい |

### グラフライブラリ: Recharts

| 観点 | 理由 |
|------|------|
| React 親和性 | React コンポーネントとして利用でき、Next.js との統合が容易 |
| 実装コスト | 円グラフ・棒グラフが少ないコードで実装可能 |

## 5. ユースケース

### UC-1: 収支を登録する
- **アクター**: ログイン済みユーザー
- **事前条件**: ログイン済みであること
- **基本フロー**:
  1. ホーム画面の「＋収支を追加する」ボタンをクリック
  2. 種別（収入／支出）を選択する
  3. 金額・カテゴリ・日付を入力し、必要に応じてメモを入力する
  4. 「登録する」ボタンをクリック
  5. ホーム画面に戻り、登録した収支が一覧に表示される
- **代替フロー**: 入力内容に不備がある場合、エラーメッセージを表示しフォームに留まる

### UC-2: 月別の収支一覧を確認する
- **アクター**: ログイン済みユーザー
- **事前条件**: ログイン済みであり、収支データが1件以上登録されていること
- **基本フロー**:
  1. ホーム画面を開く（デフォルトで当月が表示される）
  2. 「＜」「＞」ボタンで月を切り替える
  3. 選択した月の収支一覧と、収入合計・支出合計・残高が表示される

### UC-3: 収支を編集・削除する
- **アクター**: ログイン済みユーザー
- **事前条件**: ログイン済みであり、編集対象の収支が存在すること
- **基本フロー（編集）**:
  1. ホーム画面の収支一覧から対象の行をクリック
  2. 収支編集画面が表示される
  3. 内容を変更し「更新する」ボタンをクリック
  4. ホーム画面に戻り更新内容が反映される
- **基本フロー（削除）**:
  1. 収支編集画面で「削除する」ボタンをクリック
  2. 確認ダイアログが表示される
  3. 「OK」をクリックするとデータが削除されホーム画面に戻る

### UC-4: レポートでグラフを確認する
- **アクター**: ログイン済みユーザー
- **事前条件**: ログイン済みであり、収支データが1件以上登録されていること
- **基本フロー**:
  1. ナビゲーションの「レポート」をクリック
  2. 当月の支出カテゴリ別円グラフと、過去6ヶ月の収支棒グラフが表示される
  3. 月切替ボタンで表示月を変更できる

### UC-5: カテゴリを管理する
- **アクター**: ログイン済みユーザー
- **事前条件**: ログイン済みであること
- **基本フロー**:
  1. ナビゲーションの「カテゴリ」をクリック
  2. 収入・支出それぞれのカテゴリ一覧が表示される
  3. 新しいカテゴリ名を入力し「追加」ボタンをクリックするとカテゴリが追加される
  4. 既存カテゴリの編集アイコンをクリックして名前を変更できる
  5. 収支データが紐づいていないカテゴリは削除できる

### UC-6: ユーザー登録・ログインをする
- **アクター**: 未ログインユーザー
- **事前条件**: なし
- **基本フロー（新規登録）**:
  1. ログイン画面の「新規登録はこちら」リンクをクリック
  2. メールアドレス・パスワードを入力し「登録する」ボタンをクリック
  3. 登録成功後、ホーム画面に遷移する
- **基本フロー（ログイン）**:
  1. ログイン画面でメールアドレス・パスワードを入力
  2. 「ログイン」ボタンをクリック
  3. 認証成功後、ホーム画面に遷移する

## 6. 機能要件

### 6-1. 認証機能

| # | 機能 | 説明 |
|---|------|------|
| A-1 | ユーザー登録 | メールアドレス・パスワードで新規登録 |
| A-2 | ログイン | メールアドレス・パスワードでログイン |
| A-3 | ログアウト | セッションを終了する |

### 6-2. 収支管理機能

| # | 機能 | 説明 |
|---|------|------|
| B-1 | 収支登録 | 種別（収入／支出）・金額・カテゴリ・日付・メモを登録 |
| B-2 | 収支編集 | 登録済みの収支情報を編集 |
| B-3 | 収支削除 | 登録済みの収支情報を削除 |
| B-4 | 収支一覧表示 | 月別に収支の一覧を表示 |
| B-5 | 月次集計表示 | 選択月の収入合計・支出合計・残高を表示 |

### 6-3. カテゴリ管理機能

| # | 機能 | 説明 |
|---|------|------|
| C-1 | カテゴリ一覧 | 収入／支出それぞれのカテゴリを一覧表示 |
| C-2 | カテゴリ登録 | 新しいカテゴリを追加 |
| C-3 | カテゴリ編集 | カテゴリ名を編集 |
| C-4 | カテゴリ削除 | カテゴリを削除 |

**デフォルトカテゴリ（支出）**: 食費 / 交通費 / 娯楽費 / 日用品 / 医療費 / 光熱費 / 通信費 / その他  
**デフォルトカテゴリ（収入）**: 給与 / 副業 / その他

### 6-4. グラフ・レポート機能

| # | 機能 | 説明 |
|---|------|------|
| D-1 | 支出カテゴリ別円グラフ | 月別の支出をカテゴリ別に円グラフで表示 |
| D-2 | 月別収支棒グラフ | 過去6ヶ月の収入・支出・残高を棒グラフで表示 |
| D-3 | レポート画面 | グラフと月次集計をまとめたレポートページ |

## 7. 非機能要件

### 7-1. セキュリティ

| # | 項目 | 要件 |
|---|------|------|
| S-1 | 認証 | ログイン済みユーザーのみ保護ページにアクセス可能 |
| S-2 | 認可 | 自分のデータのみ参照・操作可能（他ユーザーのデータへのアクセスは403エラー） |
| S-3 | パスワード保存 | パスワードはハッシュ化して保存（平文保存禁止） |
| S-4 | 通信 | HTTPS による暗号化通信（本番環境） |
| S-5 | CSRF対策 | CSRFトークンによる対策を実施 |
| S-6 | セッション管理 | セッション有効期限を設ける（24時間） |

### 7-2. パフォーマンス

| # | 項目 | 要件 |
|---|------|------|
| P-1 | 画面表示速度 | 収支一覧・グラフ表示は3秒以内 |
| P-2 | API レスポンス | 各 API は1秒以内にレスポンスを返す |

### 7-3. 信頼性・保守性

| # | 項目 | 要件 |
|---|------|------|
| R-1 | データ整合性 | 収支削除時にカテゴリとの整合性を保つ（外部キー制約） |
| R-2 | コード管理 | Git によるバージョン管理 |

### 7-4. 対応環境

| # | 項目 | 要件 |
|---|------|------|
| E-1 | 対応ブラウザ | Google Chrome 最新版 |
| E-2 | 対応デバイス | PC のみ（レスポンシブ対応は対象外） |
| E-3 | 画面解像度 | 1280px 以上の横幅を推奨 |
